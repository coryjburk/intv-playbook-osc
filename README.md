## What this tool is

The Ops/Supply Chain Manager Interview Playbook is a **single HTML file** that runs entirely in your browser — no account, no internet connection, and no installation needed. Double-click the file and it opens.

It was built for candidates preparing for manager-level roles, with content spanning the full range from foundational concepts to expert-level judgment calls, written at the standard of a real manager-level interview.

| | |
|---|---|
| 📚 **Study** | Browse 243 pieces of content across 8 modules — questions, scenarios, frameworks, drills, and more — organized by category and difficulty. |
| 🎙️ **Practice** | Record spoken answers with a live timer and filler-word counter, or type them in. Every question, scenario, and leadership prompt has a Practice button. |
| 🤖 **Get AI coaching** | Generate a structured prompt to paste into any AI assistant (Claude, ChatGPT) and receive expert-level feedback on your answer. |
| 📊 **Track readiness** | Paste the AI's scored feedback back in and watch your competency dashboard build over time — one logged rep at a time. |

## Quick start

**Up and running in 60 seconds:**

1. **Double-click** the file `ops-supply-chain-manager-interview-playbook.html` — it opens in your default browser. No server needed.
2. Watch the **hero stats count up** in the header: 120 Questions, 30 Scenarios, 20 Frameworks, 18 Quant Drills, 20 Leadership Prompts. These confirm the file loaded correctly.
3. Click any **tab** across the top to navigate to that module.
4. Click any **card title** to expand it and read the full content.
5. When you're ready to practice, click the **blue Practice button** on any card.

> 💡 **Best browser:** Use **Chrome or Edge**. The microphone (speech-to-text) feature requires Chrome or Edge and works only on `https://` or `localhost`. If you run the file locally (a `file://` path), the mic may be blocked — type your answer instead, or [deploy to GitHub Pages](#deploying-to-github-pages) for full mic support.

## Searching and filtering

The **Questions**, **Scenarios**, and **Frameworks** tabs each have a live text-search bar — type any word or phrase and the list filters instantly on every keystroke.

Questions also have two dropdowns:

- **Category** — 12 categories including Operations Strategy, Lean/CI, Inventory, Procurement, Logistics, Manufacturing, Quality, and Supply Chain Strategy.
- **Difficulty** — see the levels below.

Scenarios have a Difficulty dropdown. Combine search with dropdowns to target exactly what you want to work on.

**Difficulty levels:**

| Level | Meaning |
|---|---|
| **Foundational** | Core concept, entry-level framing. |
| **Core** | Standard manager-level question. |
| **Advanced** | Expects structured problem-solving and trade-off reasoning. |
| **Expert** | Senior-manager judgment, ambiguity, and cross-functional complexity. |

## Reading the cards

Click any card header to expand it.

**Question cards** have four sub-tabs:

- **Conversational** — a spoken-style answer in the register you'd use with an interviewer.
- **Deep Dive** — the technical depth, with real formulas, methods, and frameworks named precisely.
- **Coaching** — the single most important signal to hit in your answer and why.
- **Red Flag** — the specific mistake that signals inexperience on this question.

**Scenario cards** have five sub-tabs:

- **Brief** — the context, the interviewer's prompt, your candidate objectives, clarifying questions to ask, and the data exhibit.
- **Approach** — the suggested structure, the key issues, and what analysis is required.
- **Answer** — the strong-answer path and final recommendation.
- **Coaching** — what a strong answer demonstrates and what the interviewer is really probing for.
- **Red Flags** — the specific mistakes that kill the score on this case.

> 💡 Use the data exhibit (a table inside the **Brief** tab) the way you would in a real case — look at the numbers and form a hypothesis *before* reading the **Answer** tab.

## Practice mode

Every question, scenario, and leadership prompt has a **Practice** button that opens a full-screen modal for a complete rep.

Inside the modal:

| | |
|---|---|
| ⏱️ **Timer** | Starts at 00:00 when you begin recording. Helps you stay within a realistic 2–4 minute answer. |
| 🎤 **Mic / Recording** | Click the mic to start speech-to-text; click again to stop. Requires Chrome or Edge. Degrades gracefully to a text box if the mic is unavailable. |
| 🚫 **Filler-word counter** | Updates live as you speak or type. |
| 📝 **Transcript box** | Your spoken words appear here automatically. You can also type directly. Text is editable — clean it up before generating the coaching prompt. |

**Tracked filler words:** `um` · `uh` · `like` · `you know` · `kind of` · `sort of` · `basically` · `actually` · `literally` · `I mean` · `right`

**Recording your answer:**

1. Click **Start recording**. The mic button turns red and the timer starts.
2. Speak your answer naturally, as if you're in the room with an interviewer.
3. Click **Stop recording** when done. Fix any speech-to-text errors in the transcript.
4. Review the filler-word count — aim for **fewer than 5** in a 2-minute answer.
5. If the mic isn't available, just type your answer in the transcript box.

> ⚠️ **Mic not working?** The Web Speech API requires (a) Chrome or Edge and (b) an `https://` URL or `localhost`. On a `file://` path, allow microphone access in the browser's site-settings panel, or [deploy to GitHub Pages](#deploying-to-github-pages) for a clean https setup.

## AI coaching loop

Once you have a transcript (typed or spoken), click **Generate coaching prompt**. A full prompt appears — pre-loaded with the question, your answer, and a rubric asking the AI to evaluate eight competencies.

```
Record answer → Generate prompt → Copy prompt → Paste into Claude / ChatGPT → Read feedback + scores → Log rep to dashboard
```

Click **Copy prompt**, open Claude (claude.ai) or ChatGPT, and paste it in. The AI returns written feedback plus a machine-readable score block at the end.

## Readiness dashboard

The **Readiness** tab turns practice into measurable progress, built from the AI coaching scores you paste in across every rep you log.

**The eight competency dimensions:**

- Problem structuring & root-cause rigor
- Operational & technical knowledge
- Quantitative & metrics fluency
- Process-improvement application (Lean/Six Sigma)
- Trade-off & business judgment
- Leadership & people management
- Stakeholder communication & executive presence
- Recommendation & implementation focus

Each dimension is scored 0–100 per rep. The dashboard shows your running average per dimension and your overall average across all scored reps.

**How to log a rep:**

1. Complete a practice rep and paste the coaching prompt into Claude or ChatGPT.
2. Read the written feedback the AI returns.
3. Scroll to the bottom of the reply — you'll see a `[READINESS SCORES]` block.
4. Select and **copy the entire AI reply** (including the score block).
5. Go to the **Readiness** tab and paste the text into the **Log a rep** box.
6. Click **Log this rep**. The dashboard updates immediately.

The score block at the bottom of the AI's reply looks like this:

```
[READINESS SCORES]
Problem structuring & root-cause rigor: 74
Operational & technical knowledge: 81
Quantitative & metrics fluency: 66
Process-improvement application (Lean/Six Sigma): 70
Trade-off & business judgment: 78
Leadership & people management: NA
Stakeholder communication & executive presence: 72
Recommendation & implementation focus: 80
[END READINESS SCORES]
```

> 💡 **Copy the full reply** — the parser finds the block automatically. You don't need to extract just the score lines.

**Readiness levels:**

| Level | Overall score | Meaning |
|---|---|---|
| **Emerging** | < 45 | Early reps — focus on structure and fundamentals. |
| **Developing** | 45–67 | Real progress — close the weak competencies. |
| **Interview-Ready** | 68–84 | Solid and consistent — keep sharpening. |
| **Manager-Caliber** | ≥ 85 | Performing at the bar for the role. |

**Dimensions scored `NA`:** If the AI marks a dimension `NA` (the question didn't exercise it), that rep doesn't count toward that dimension's average. Only reps where a dimension was actually scored 0–100 contribute — keeping the dashboard honest.

## Saving, exporting & importing progress

Logged reps and saved notes are written to your browser's **localStorage**, which persists across sessions on the same browser and device.

The Readiness tab has three data-management buttons:

- **Export progress** — downloads a JSON file (`ops-interview-progress.json`) with all your logged reps and saved notes. Use it to back up or move to another device.
- **Import progress** — loads a previously exported JSON file and merges the data into your current session.
- **Reset** — clears all logged reps and saved notes after a confirmation prompt. **Permanent and cannot be undone.**

You can also **save a note** per question inside the Practice modal — your transcript saves to that specific question and reloads next time you open it.

> ⚠️ **Export before switching browsers.** Progress lives in browser storage. If you clear browser data, switch browsers, or use a private/incognito window, your data won't be there. Export regularly if you're building a history you want to keep.

## Suggested workflow

**Phase 1 — Ground yourself (Days 1–3).** Skim all 15 **Battlecard** cards. Read through **Frameworks** and **Quant Drills**. Check the **Red Flags** tab so you know the mistakes before you make them.

**Phase 2 — Work the questions by category (Days 4–10).** Use the Category filter in **Questions**. Read Conversational and Deep Dive first, then **Practice**, record, generate the coaching prompt, and log the rep. Do Foundational and Core before Advanced and Expert.

**Phase 3 — Run the cases (Days 8–14).** Open a **Scenario**, read only the **Brief**, then close it and talk through your answer out loud for 3–5 minutes *before* expanding Approach and Answer. Forming your own hypothesis first is where the learning happens.

**Phase 4 — Sharpen weak dimensions (Ongoing).** After 10+ logged reps, your dashboard shows a pattern. Target your two weakest competencies with specific questions and scenarios.

**Phase 5 — Pre-interview day.** Review the **Battlecard** power phrases and 30/60/90-day card, run the **Red Flags** checklist, and warm up with 2–3 Leadership prompts in practice mode. Your current Readiness level is your honest baseline.

> 💡 **One rep a day beats a weekend marathon.** The competency averages are most useful when built from varied questions across categories. Spread practice across all eight tabs over days, not hours.

## Deploying to GitHub Pages

Hosting the tool on GitHub Pages gives you a clean `https://` URL — which enables the microphone and lets you share it.

1. Create a new repository and add the HTML file, renaming it **`index.html`**.
2. Go to **Settings → Pages**.
3. Set **Source** to *Deploy from a branch*, **Branch** to `main`, folder to `/ (root)`, and click **Save**.
4. The tool goes live at `https://<your-username>.github.io/<repo-name>/` within about a minute.

Microphone and progress-saving both work normally on GitHub Pages.

## FAQ

<details>
<summary><strong>Does this tool require an internet connection?</strong></summary>

No. The HTML file is fully self-contained — all content, JavaScript, and styles are embedded, and it works offline after the first load. The only features needing connectivity are loading the web fonts on first open (they fall back gracefully if offline) and the AI coaching step, which needs a browser tab open to Claude or ChatGPT.
</details>

<details>
<summary><strong>My microphone isn't working. What should I do?</strong></summary>

The speech recognition uses the browser's Web Speech API, which requires Chrome or Edge (not Firefox or Safari) and a page served over `https://` or `localhost`. A local `file://` path often blocks the mic. The simplest fix is to [deploy to GitHub Pages](#deploying-to-github-pages) and use it from there. In the meantime, just type your answer — the coaching workflow is identical either way.
</details>

<details>
<summary><strong>Will my progress be saved if I close the browser?</strong></summary>

Yes — logged reps and saved notes are stored in your browser's localStorage and persist across sessions on the same browser and device. Export your progress as a JSON backup before clearing any browser data.
</details>

<details>
<summary><strong>Can I use this on my phone or tablet?</strong></summary>

Yes — the tool is responsive. The tab bar scrolls horizontally on narrow screens, the transcript box is touch-editable, and the dashboard works normally. Speech recognition on mobile Chrome works if the file is served over `https://`.
</details>

<details>
<summary><strong>Which AI assistant should I use for coaching?</strong></summary>

Any capable assistant works — Claude (claude.ai), ChatGPT, or Gemini. Claude tends to produce the most structured, candid operations-specific feedback. Paste the generated prompt as a new message, let it finish, then copy the entire reply (including the `[READINESS SCORES]` block) back into the Readiness tab.
</details>

<details>
<summary><strong>The AI didn't include a score block. What happened?</strong></summary>

Occasionally an assistant truncates or reformats long replies. If the block is missing, add: *"Please include the [READINESS SCORES] block at the very end of your reply, in exactly the format specified in the prompt."* You can also type a score block manually — the parser accepts any line with a competency keyword followed by a colon and a number or `NA`.
</details>

<details>
<summary><strong>Can I print the Battlecards or Red Flags?</strong></summary>

Yes. Use your browser's Print function (<kbd>Ctrl</kbd>/<kbd>Cmd</kbd> + <kbd>P</kbd>) while on the Battlecard or Red Flags tab. Built-in print styling removes navigation and renders only the active panel. Print to PDF for a take-anywhere reference.
</details>

<details>
<summary><strong>I scored low on one dimension. What should I practice?</strong></summary>

Each competency maps to specific content:
- **Quantitative fluency** → Quant Drills + Lean/CI and Manufacturing question categories
- **Problem structuring** → all Scenario cases + Process Improvement questions
- **Technical knowledge** → Operations Strategy, Inventory, Procurement, and Logistics categories
- **Leadership & people management** → Leadership tab
- **Judgment & trade-offs** → Expert-difficulty scenarios and questions

Use the Category and Difficulty filters to target exactly where you need reps.
</details>
